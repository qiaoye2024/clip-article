# clip-fetch.sh — 微信公众号文章抓取脚本

> **版本**: v4.4 | **行数**: 1029 | **最后更新**: 2026-04-08
>
> **路径**: `~/.claude/skills/clip-article/clip-fetch.sh`（与 `Notes/obsidian-system/.claude/skills/clip-article/clip-fetch.sh` 硬链接同步）

---

## 一、功能概述

通过 **Claudian CDP 浏览器** 打开微信公众号文章 URL，提取：

- **元数据**：标题、作者、发布日期
- **正文**：HTML → Markdown 转换（含标题、列表、引用、代码块等）
- **图片**：下载到本地 Obsidian vault 的 `04-assets/images/` 目录
- **图注 (caption)**：自动从 DOM 结构中识别每张图片的说明文字

最终输出 **结构化 JSON** 到 stdout，供 AI（SKILL.md 指引下的 Claude/GPT）生成 Obsidian 笔记。

---

## 二、用法

```bash
bash clip-fetch.sh <微信公众号URL> [Vault目录]

# 示例
bash clip-fetch.sh "https://mp.weixin.qq.com/s/xxx" "/Users/linlang/Notes/obsidian-system"
```

| 参数 | 必填 | 说明 | 默认值 |
|------|------|------|--------|
| `URL` | ✅ | 微信公众号文章完整 URL | — |
| `[输出目录]` | ❌ | Obsidian vault 根目录（用于存放图片） | 当前工作目录 (`pwd`) |

### 环境变量

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `CDP_PORT` | `3456` | CDP Proxy HTTP API 端口 |

### 输出格式

JSON 输出到 stdout，包裹在标记之间便于解析：

```
========== OUTPUT_JSON_START ==========
{... JSON ...}
========== OUTPUT_JSON_END ==========
```

任何退出路径（成功/失败/异常）都保证有 JSON 输出（`trap cleanup EXIT`）。

### 成功时 JSON 字段

```json
{
  "title": "文章标题",
  "author": "作者名",
  "publishDate": "2026-04-02",
  "source": "原始URL",
  "text": "Markdown 正文，含 {{IMG_N}} 占位符",
  "imgDir": "/绝对路径/04-assets/images/短标题",
  "relDir": "04-assets/images/短标题",
  "images": [
    {
      "idx": 0,
      "filepath": "/绝对路径/图片.png",
      "filename": "图片.png",
      "caption": "图注文字",
      "markdown": "![[04-assets/images/短标题/图片.png]]"
    }
  ],
  "allImages": [ /* 全部图片（含失败的），与 {{IMG_N}} 索引一一对应 */ ],
  "failed": [ { "idx": N, "src": "...", "caption": "" } ],
  "downloadedCount": 12,
  "failedCount": 0
}
```

> **关键字段 `markdown`**（v4.4 新增）：包含完整的 Obsidian 图片嵌入语法 `![[relDir/filename]]`。
> AI 生成笔记时必须原样使用此字段，禁止自行拼路径或修改字符。

### 失败时 JSON 字段

```json
{
  "error": "人类可读的错误信息",
  "errorType": "cdp_unavailable | tab_creation_failed | meta_extract_failed | content_extract_failed | unexpected_exit"
}
```

```

## 四、核心算法详解

### 4.1 Caption 图注提取（第 199~349 行）

#### 多维打分模型 `scoreCaptionCandidate()`

替换了旧版的枚举词表法（`isNarrativeText`），采用**特征打分 + 阈值判定**：

**基准分**: 50 分。≥ 60 判定为图注，< 60 判定为正文。

| 因子 | 条件 | 加减分 | 思路 |
|------|------|--------|------|
| **长度** | 4~20 字 | +30 | 图注通常短 |
| | ≤35 字 | +15 | |
| | >50 字 | -40 | 长文字大概率是正文 |
| **句号数** | 0 个句号 | +20 | 图注一般不是完整句子 |
| | 1 个句号 | +5 | |
| | ≥2 个句号 | -30 | 多句=叙述性正文 |
| **图注关键词开头** | 以"图:"/"图片:"/"来源:"等开头 | +40 | 强信号 |
| **特殊符号开头** | ▲/△/↑/【图】/【注】 | +35 | 明确的图注标记 |
| **摄影相关词** | 含"摄影"/"拍摄"/"截图"等 | +25 | 典型图注词汇 |
| **叙述性开头词** | 这/那/他/她/我/你...且长度>15字 | -35 | 叙述性正文典型开头 |
| **时间词开头** | 去年/今年/202X年/此前... | -30 | 时间线叙述非图注 |
| **转折连词开头** | 但是/然而/不过/因此... | -40 | 论证性连接词 |
| **动词+语气词模式** | 中文{2,+}了/着/过/起来+标点且>20字 | -25 | 动作描述 |

#### 8 条 Caption 提取路径（优先级从高到低）

| 路径 | DOM 结构 | 适用场景 |
|------|----------|---------|
| **路径1** | `FIGURE > figcaption` | 标准 HTML5 语义标签 |
| **路径2** | img.nextElementSibling (FIGCAPTION/P/SPAN/text) | 微信最常见：图片紧跟文字 |
| **路径3** | 父元素内 img 之后的兄弟节点 | `<p>img</p><p>图注</p>` |
| **路径4** | 向上遍历父链找相邻段落 |深层嵌套如 `section>p>img` |
| **路径5** | img.previousElementSibling (P, ≤30字) | 图注在图片上方 |
| **路径6** | `img.alt` 属性 | 最后兜底 |
| **路径7** | 父元素内 img 后的 text node | `<p>图片<text>说明</text></p>` |
| **路径8** | 相邻 section 容器的后续兄弟 | section 内独立文本 |

> 所有路径都经过 `isLikelyCaption()`（即 `scoreCaptionCandidate() ≥ 60`）验证后才采纳。

### 4.2 HTML → Markdown 转换器（第 366~657 行）

递归树遍历转换器 `htmlToMarkdown(node)` + `processChildren(node)`。

支持的标签转换：

| HTML 标签 | Markdown 输出 |
|-----------|--------------|
| H1-H6 | `#` ~ `######` + 标题文字 |
| BLOCKQUOTE | `>` 引用块（按行包裹） |
| UL/OL | `-` / `1.` 列表 |
| PRE/CODE | `` ``` `` 代码块 |
| HR | `---` 分割线 |
| SECTION | 启发式标题检测（见下方）/ 递归子节点 |
| P | 普通段落 + 章节标题检测（BUG-6）/ `\n\n` |
| SPAN | 斜体/下划线/删除线（基于内联样式） |
| STRONG/B | `**加粗**` |
| EM/I | `*斜体*` |
| A | `[文字](URL)` |

#### SECTION 启发式标题检测（第 491~517 行）

当 SECTION 满足以下任一规则时，渲染为 `### 标题`：

- **规则A**: 字号 ≥ 18px + 加粗 + 文字 < 30 字
- **规则B**: 字号 ≥ 16px + 加粗 + 文字 ≤ 5 字（纯编号如 "1" "2"）
- **规则C**: 字号 ≥ 20px + 独占一行（前后兄弟无文本）+ 文字 < 40 字

#### P 标签章节标题检测（BUG-6，第 520~574 行）

针对微信编辑器特有的 **FONT > SPAN(bold) > SPAN(red)** 嵌套结构：

```
<P>
  <FONT>
    <SPAN style="font-weight:bold">
      <SPAN style="color:#ab1942">03. 偏安的结构</SPAN>
    </SPAN>
  </FONT>
</P>
```

检测条件（全部满足）：
1. P 的直接子元素是 FONT 或单个 SPAN
2. 内层包含红色 SPAN（色值含 `171` / `ab1942` / `red`）
3. 或外层 SPAN 本身为 bold
4. 文本匹配 `^(0?[1-9])\.\s+` 编号格式
5. 总文字 < 30 字

### 4.3 图片过滤规则（第 169~196 行）

| 过滤类型 | 检测方式 | 说明 |
|----------|----------|------|
| **GIF 动图** (BUG-5) | URL 路径含 `/mmbiz_gif/` 或以 `.gif` 结尾 | 微信 GIF 几乎都是装饰/推广动画 |
| **data: URI** | `src.startsWith("data:")` | 内嵌数据非网络图片 |
| **视频封面/视频内图片** | 向上遍历查 VIDEO 标签或 `data-vid` 属性 | 视频组件内的图 |
| **微信 UI 垃圾** | alt/class/文件名/后续兄弟文本含关键词 | 关注按钮、分享、点赞等 |
| **公众号卡片区域** | `closest('#mp_name_icon, .rich_media_tool, .reward_pannel, .article_bar...')` | 底部栏区域 |

垃圾关键词列表：`已关注/关注/分享/赞/在看/更多/退出全屏/切换到/倍速播放/播放进度/全屏/您的浏览器不支持/观看更多/珍爱包叔/写下你的评论`

### 4.4 文本后处理（第 663~747 步）

按顺序执行：

| # | 处理 | 说明 |
|---|------|------|
| 1 | 多余空行合并 | `\n{3,}` → `\n\n` |
| 2 | 列表项间空行清理 | 列表项之间不应有空行 |
| 3 | **去重** | 连续相同行（忽略 `>` 前缀）只保留一份 |
| 4 | 空 `****` 清理 | 空 strong/em 嵌套产生的残留 |
| 5 | 段首/段尾/中间粘连 `****` 清理 | 各种位置的空加粗标记 |
| 6 | **底部栏截断** | 从"责任编辑"位置 substring 截断 |
| 7 | 底部关键词截断 | "更多精彩"/"微信扫一扫"/"阅读原文" |
| 8 | **"全文完"删除** (BUG-3) | 正则清除公众号结束语 |
| 9 | **空标题删除** (BUG-4) | 清除残留的 `#####` |
| 10 | **推广块精准删除** (BUG-2) | 定位"珍爱包叔"等推广语，向前找占位符+向后遇正文图片停止 |
| 11 | 豆包AI播客行清除 | `"此节目由豆包AI播客生成..."` |
| 12 | 最终空行合并 | 再清一遍 `\n{3,}` |

## 五、图片下载机制（第 855~951 行）

### 双重下载策略

```
curl 下载（主）
  ├─ HTTP 200 + 文件大小 > 500B → ✅ 成功
  └─ 失败（HTTP 错误/超时/文件太小）
      └─ fetch + base64 兜底（通过 CDP 浏览器 JS 执行 fetch API）
          ├─ base64 数据有效 + 解码后 > 500B → ✅ 成功
          └─ 失败 → ❌ 标记 failed
```

### curl 参数

```bash
curl -s -o "$FILEPATH" -w "%{http_code}" "$IMG_SRC" \
  -H "Referer: https://mp.weixin.qq.com" \   # 微信 CDN 需要 Referer
  --max-time 15 \                               # 单次超时 15 秒
  --retry 1 \                                   # 失败重试 1 次
  --retry-delay 2 \                             # 重试间隔 2 秒
  --retry-all-errors                           # 所有错误码都重试（不仅 5xx）
```

### 文件名生成规则

```
[短标题] caption文字前10位.扩展名     ← 有 caption 时
[短标题] 图片上下文_序号.扩展名        ← 无 caption 时
```

- **短标题**：取文章标题前 10 字符，去除副标题（—/｜之后）、空格、特殊字符
- **扩展名**：优先从 URL 参数 `wx_fmt=` 提取，默认 `jpg`
- **重名处理**：自动追加 `_2` `_3` ... 递增

### 文件存放位置

```
{Vault目录}/04-assets/images/{短标题}/[短标题] xxx.png
```

例如：`/Users/linlang/Notes/obsidian-system/04-assets/images/BBA车主，加速流向/[BBA车主，加速流向] 智己LS8.png`

## 六、工程细节

### 6.1 安全机制

| 机制 | 实现位置 | 说明 |
|------|----------|------|
| **trap cleanup EXIT** | 第 18 行 | 任何退出（包括 Ctrl+C、set -e、未捕获错误）都保证输出 JSON |
| **safe_eval() 超时保护** | 第 86~108 行 | 每次 JS eval 有超时参数，默认 30 秒；大段 JS 用临时文件传递 |
| **safe_eval 空值检查** | 第 101~106 行 | 返回 null/空时标记失败，静默失败比报错更危险 |
| **CDP 3 次重试** | 第 29~37 行 | 连接失败自动重试，每次间隔 2 秒 |
| **双重 JSON 校验** | 第 141/764 行 | 元数据和正文都校验是否为合法 JSON |
| **curl → fetch 兜底** | 第 900~949 行 | curl 下载失败时用浏览器 fetch API 转 base64 兜底 |

### 6.2 大数据处理方案

| 问题 | 方案 |
|------|------|
| JS 代码超出 HTTP POST 参数限制 | 写入临时 `.js` 文件，用 `--data-binary @file` 传递（第 160 行） |
| TEXT_BODY 超出 jq 命令行参数限制 | 写入临时文件，用 Python 读取并组装最终 JSON（第 964~1017 行） |
| 文件名含中文特殊字符 | 用 Python3 处理 UTF-8 安全字符（第 790~806 行） |

### 6.3 依赖清单

| 依赖 | 版本要求 | 用途 |
|------|----------|------|
| **bash** | 3.x+ | 脚本运行环境 |
| **curl** | 7.x+ | HTTP 请求（CDP API + 图片下载） |
| **jq** | 1.6+ | JSON 处理 |
| **python3** | 3.6+ | 标题安全处理 + 最终 JSON 组装 |
| **Claudian Obsidian 插件** | — | 提供 CDP HTTP API（localhost:3456） |
| **stat** | BSD/macOS | 检查文件大小（`stat -f%z`） |
| **base64** | 系统自带 | 解码 fetch 兜底的 base64 图片数据 |

> ⚠️ **平台兼容性注意**：`stat -f%z` 是 macOS/BSD 语法，Linux 上需改为 `stat -c%s`。

## 十、调试技巧

### 查看脚本实际输出的 JSON

脚本输出到 stdout，Claudian 会捕获并传给 AI。如果想手动查看：

```bash
bash clip-fetch.sh "https://mp.weixin.qq.com/s/xxx" "/path/to/vault" 2>/dev/null \
  | sed -n '/OUTPUT_JSON_START/,/OUTPUT_JSON_END/p' \

```

### 只测试某一步

可以在脚本中对应步骤前后加 `exit 0` 来提前终止，查看中间结果。

### 常见错误排查

| 错误信息 | 原因 | 解决方法 |

| `CDP 连接失败` | Claudian 浏览器模式未启用 | Obsidian 中开启 Claudian，确认端口 3456 可访问 |
