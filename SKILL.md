## 触发条件

用户输入包含 URL 时立即激活，无需额外指令。

## 流程

### 第一步：运行脚本

```bash
cd /Users/linlang/Notes/obsidian-system
bash .claude/skills/clip-article/clip.sh "URL" "/Users/linlang/Notes/obsidian-system"
```

脚本会自动完成：抓取文章 → 下载图片 → 替换图片占位符 → 生成 frontmatter → 写入 .md 文件。

**⚠️ 关键：必须从脚本的 stdout 输出中提取生成的文件路径！** 禁止自己猜测或编造文件名。脚本成功后，stdout 中会包含 `📝 文件路径：` 行，后面的路径就是实际生成的 .md 文件绝对路径。后续第三步读取文件时必须使用这个路径。

**输出情况：**
- ✅ 成功：stdout 显示 `✅ 笔记已生成` + `📝 文件路径：/绝对/路径/文件名.md` + 图片统计
- ⚠️ 文件已存在：stdout 显示 `FILE_EXISTS:路径`，需要询问用户是否覆盖
- ❌ 失败：stderr 显示错误信息，告知用户并停止

### 第二步：文件已存在时（仅当脚本输出 FILE_EXISTS）

告知用户文件路径，询问是否覆盖。确认后删除旧文件，重新运行第一步。

### 第三步：AI 填充理解字段（必须执行！脚本输出后任务未结束）

读取脚本生成的 .md 文件，找到所有 `<!-- FILL:xxx -->` HTML 注释标记，替换为实际内容。

共 **5 个理解型字段**，必须全部填充，一个都不能跳过：

#### 1. tags（2~4 个标签）

生成规则（按优先级）：
1. 先看标题和正文是否有明确领域关键词（如：AI/经济/房产/营销/创业/管理/技术），选最贴的 2~3 个
2. 加 1 个固定标签：`好文章`
3. 如果作者是有影响力的账号（如任泽平、吴晓波、刘润等），加 1 个 `人/作者名` 标签
4. 总数 2~4 个，宁少毋滥。禁止泛泛标签如"文章""阅读""推荐"
5. **格式必须是 YAML 列表**，每行 `- 标签名`，禁止用任何逗号：
   ```
   tags:
   - AI
   - 好文章
   - 任泽平
   ```

#### 2. summary（1~3 句中文概括）

- 用一句话说清楚这篇文章讲了什么 + 一个核心观点或结论
- 不要抄标题，要读完正文后的真正概括
- 不超过 100 字
- 示例格式："本文分析了XXX，指出YYY，认为ZZZ。"

#### 3. ==关键要点==

- 基于理解自由总结，用自己的话写
- 要点列表形式，3~7 条

#### 4. ==我的思考==

- **第一人称**，以一个真实读者的身份读完文章后写的直接反应
- 可以是：对某个观点的赞同或反驳、联想到的类似现象、觉得哪里没说透、读完最想记住的一句话
- 3~5 条，口语化，有情绪，不说套话
- **边界**：这里写的是"我觉得"，是读者的主观感受——知识层面的延伸留给"发芽"

#### 5. ==发芽==（跨领域启发）

这是本技能最有价值的部分——从文章的思想内核出发，跳出原文领域找知识连接。

**整体结构**：
1. **引言**（1~2句）：用"你今天不仅是在…更是在追问…"的句式，点明这篇文章在更大视野下的意义
2. **3~4 颗芽**，每颗格式：

```
**编号. 主题标题**

🌱 种子
从原文某个具体论点出发，引入一个跨域的知识参照——
可以是历史人物、思想理论、社会学概念、文学作品、跨行业案例。
要求：不复述原文，要超出原文范围，找到真正的知识连接点。

✨ Aha 瞬间
"一句话金句，浓缩这颗芽的核心洞见，带引号"
```

3. **金句回响**：从哲学、文学、社会学三个视角各引用一句经典名言，与原文主题形成共鸣

**关键要求**：
- 每颗芽必须跨域——从文章领域跳到历史/哲学/科学/其他行业
- Aha 瞬间必须是一句带引号的金句，让人读完有"原来如此"的感觉
- 质量 > 数量

### 第四步：保存并告知

```
✅ 文章已处理完成

**文件**：[[02-Knowledge/好文章/作者名 文章标题]]
**图片**：04-assets/images/短标题/ (成功 N 张，失败 M 张)
```

失败图片额外告知 URL。

## 五条铁律

1. **URL 即命令。** 检测到 URL 必须使用本技能
2. **脚本失败 = 任务失败。** 不许用替代方案自行获取数据
3. **只填 5 个理解字段。** tags、summary、关键要点、我的思考、发芽——找到 `<!-- FILL:xxx -->` 注释标记替换为实际内容，其他一律不改。一个都不能跳过！
4. **完整原文一个字不动。** 只填充留空区域，不得修改脚本生成的 frontmatter 机械字段和正文内容
5. **文件存在先询问。** 须告知路径并询问覆盖，不得静默跳过或擅自覆盖
6. **必须用脚本输出的路径读取文件。** 禁止自己猜测、编造或从历史会话记忆中取文件名。第三步读取 .md 文件时，只能使用脚本 stdout 中 `📝 文件路径：` 后面的绝对路径

## ⚠️ 禁止行为

- **禁止调用 web_fetch / WebFetch / any URL fetching tool**：本技能的数据获取完全依赖 `clip-fetch.sh` 脚本（CDP 浏览器通道）
- **禁止修改脚本生成的 frontmatter 机械字段**（title/source/author/published/created）
- **禁止修改完整原文区域**（包括图片嵌入语法、章节标题、正文格式）
